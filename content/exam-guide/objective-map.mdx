# AB-900: Microsoft 365 Copilot and Agent Administration Fundamentals

To pass the exam, you must master these three core domains. This portal is now mapped 1:1 to these objectives.

## 1. Core Features and Objects (30–35%)
- [ ] **License Types:** How license assignments affect feature access.
- [ ] **M365 Admin Center:** Domain names and organization settings.
- [ ] **Exchange Online:** Mailboxes and distribution lists configuration.
- [ ] **SharePoint Admin:** Sites, libraries, folders, and permissions.
- [ ] **Teams Admin Center:** Teams, channels, and policy management.
- [ ] **Security Principles:** Zero Trust, Authentication vs. Authorization.
- [ ] **M365 Security Features:** Microsoft Entra ID, Conditional Access, SSO, and PIM.

## 2. Data Protection and Governance (35–40%)
- [ ] **Microsoft Purview:** Information Protection, DLP, Insider Risk, and Data Lifecycle.
- [ ] **Sensitivity Labels:** Use cases and classification.
- [ ] **Copilot Security:** Data access via Microsoft Graph, Responsible AI principles.
- [ ] **Risk Identification:** Compliance Manager, Data Explorer, and Activity Explorer.
- [ ] **SharePoint Oversharing:** Data access governance reports and Restricted Site Access.

## 3. Administrative Tasks for Copilot & Agents (25–30%)
- [ ] **Capabilities:** Built-in Copilot features vs. custom agents.
- [ ] **Licensing Models:** Monthly license vs. Pay-as-you-go (PAYG).
- [ ] **Admin Tasks:** Assigning licenses, monitoring usage, and managing prompts.
- [ ] **Agent Lifecycle:** Creating, approving, monitoring, and managing agents in Power Platform.
