# Microsoft Purview & AI Governance

In the AB-900 exam, Governance accounts for **35-40%** of the score. This is the most critical section for production readiness.

## 1. Sensitivity Labels (The "Eyes" of the Agent)
Sensitivity labels define what data an AI Agent or Copilot can "see" and "cite."

*   **How it works:** If a document is labeled "Confidential," the AI agent will only process it if the user asking the question has the appropriate permissions.
*   **Exam Tip:** Sensitivity labels do **not** override existing SharePoint permissions; they add an extra layer of protection.

## 2. Data Loss Prevention (DLP) for Copilot
DLP policies prevent sensitive information (like credit card numbers or NRICs) from being shared in AI conversations.

*   **Blocking:** You can set a policy that blocks Copilot from responding if the source document contains sensitive info types (SITs).
*   **Auditing:** Even if not blocked, every interaction is logged for compliance review.

## 3. Retention vs. Protection
*   **Retention:** Defines how long AI conversation history is *kept* (for legal discovery).
*   **Sensitivity:** Defines who can *access* the data.
*   **The Trap:** AB-900 often asks which one to use for "data leakage" (Sensitivity/DLP) vs "legal compliance" (Retention).

## 4. Audit Logs
Every prompt and every AI response is stored in the **Unified Audit Log**.
*   **Admin Center:** Access this via the Microsoft Purview compliance portal.
*   **Use Case:** Investigating "Hallucinations" or "Inappropriate content" reported by users.
