# Zero Trust & Security Principles

Microsoft 365 security is built on the **Zero Trust model**.

## 1. Zero Trust Principles
"Never trust, always verify."
- **Verify Explicitly:** Always authenticate and authorize based on all available data points (identity, location, device, data).
- **Least Privilege Access:** Limit user access with Just-In-Time (JIT) and Just-Enough-Access (JEA).
- **Assume Breach:** Minimize blast zones and segment access. Verify end-to-end encryption.

## 2. Authentication vs. Authorization
- **Authentication (AuthN):** Proving *who* you are (Password, MFA, SSO).
- **Authorization (AuthZ):** Determining *what* you can do (Roles, Permissions, RBAC).

## 3. Microsoft Entra & Conditional Access
Conditional Access is the "Policy Engine" for Zero Trust.
- **Signals:** User, Location, Device, Application, Risk level.
- **Decision:** Allow, Block, or Require MFA.
- **Enforcement:** Grant access only when conditions are met.

## 4. Single Sign-On (SSO)
- **Benefit:** One set of credentials for all apps.
- **Goal:** Improve security and reduce password fatigue.
